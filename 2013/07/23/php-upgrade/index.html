PHP每次升级都升级了什么？

前言

我是2010年中旬开始自学PHP的，那时候大家用的PHP版本还是5.2甚至是5.1,这几年一直在学PHP，可以说是见证了从PHP 5.2到PHP 5.5的过程。不怕大家笑话，那时我看个 MVC 都非常的困难。那时多菜呢，学了半年之后的也就是2011年初，还在看MVC，一个 《最简单的PHP_MVC留言本实例.doc》看了又看实在是看不懂。不解啊，困惑啊，后来突飞猛进的一天是就业实习期间跟 柒零叁 的技术负责人有过一次交道，然后照猫画虎的按照他的路子写了一个MagicPHP，在这期间深入学习了MVC（其实真说起来还不是完全意义上的MVC只是类似。），自认为从那天起PHP才算是入门。那时的代码写得是非常的烂，按照今天的标准要是我组里谁给我发现写出这种代码，绝对会给我吐槽到死的。可是就在当时703的技术负责人邀请我进入703，我觉得我的代码风格起到了一定因素，所以我至今都认为 “没有能力不行的只有天赋不够或者态度不行的”。

有点扯远了，今天说的是“PHP每次升级都升级了什么”,我觉得将我的 PHP 背景代入下应该更有说服感。

我觉得 PHP 每次升级都是围绕着几个大的功能点来的，哪几个呢？我列一下:

一、“更安全”

其实这是个伪命题，因为安全这一块往往都不掌握在 PHP 语言开发者的手中，而是 PHP 代码编写者的手中，然后为什么要放在这里呢？这得从 PHP 的 “safe mode“ 跟 “magic quote“ 说起，这对难兄难弟都恰好是在 PHP 5.3.0 起废弃并将自 PHP 5.4.0 起移除。关于安全模式官方已经很好的说明了废弃的理由:

二、“更高效”

更高效这里应该指的是两个方面，一个是 PHP 执行代码的速度更高效，第二个是开发者开发的速度更高效。

首先讲第一个方面:代码高效最简单的体现方式就是执行速度更快已经内存占用更小，我没有实际接触过这两方面的benchmark。但是根据网上得来的评价[1] [2] [3]来看:PHP的每一个升级版本相比于上一个版本都是有轻微的性能提升，无论是在执行速度上还是内存占用上。

捡几个重点的讲一下：

  • JSON extension was added and enabled by default. 这个特性使得我们无需在自行加载第三方 JSON 解析器，只需使用自带方法json_encode,json_decode就可以了。

  • Added lambda functions and closures. 这个特性可以见我的文章: PHP中的闭包。

  • Added support for namespaces. 这个特性我觉得很重要，写Class的时候终于不需要再加上长长的Package前缀了。

  • Added DIR constant. 这个特性看起来不是很重要，但是它不知道替代了多少的dirname(__FILE__)。

  • Added support for Traits. 这个特性也可以见我的文章： PHP traits 使用范例

  • Built-in webserver in CLI mode to simplify development workflows and testing 这个使得初学者在开发的时候终于不用再配置Web环境了。

  • Added the Zend OPcache extension for opcode caching. 这个使得APC都没用用武之地了。

因为涉及到的特性非常的多，我就不一一解释了，有兴趣的自行Google。

三、“更人性化”

人性化似乎更PHP的升级一点都没关系，但其实不然。因为很多时候PHP的新特性并不是凭空产生的，而是经过广大用户的讨论以及PHP核心开发组的投票。比如这个RFC：Request for Comments: Generators 的建立时间是2012-06-05，但直到最近的PHP 5.5.0才去实现，你可以从 PHP RFC 这个站点去了解各种各样的RFC，事实上你可以从RFC中看出PHP下一个版本的大致样子。

除了PHP自己的RFC，PHP同时也会实现一部分的 IETF的RFC，以使得PHP更符合世界标准。
