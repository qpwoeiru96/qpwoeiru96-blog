使用Node.js进行安全的翻墙

最近 GFW 发病发的厉害导致开发效率大幅度降低，虽然本人有 VPS 护驾，但是通过 SSH 进行代理的方式并不高效，经常出现重连的情况。

苦苦寻觅终于在 Github 上找了一个类似 GoAgent 的解决方案：node-spdyproxy。

node-spdyproxy 有以下优点：

  1. 非透明代理方式，更安全。
  2. SPDY 连接方式，效率更好。
  3. 跨平台支持 Windows、*unix。
  4. 安装方便、无需编译。

当然唯一不适应的就是如果没有经过认证的证书那么需要在每台 Client 上都安装一下证书。

安装过程：

  • 首先：安装 Node.js （这一环节就不再赘述）。
  • 使用 npm -g install spdyproxy 安装 node-spdyproxy 。
  • 安装完成之后，进入公共的 node_modules 文件夹（一般是在 /usr/local/lib/node_modules ），进入 spdyproxy 文件夹，执行 mocha 看一下是否测试通过。

如果通过表明你成功的安装了 spdyproxy ，接下来便是配置安全证书的过程。当然如果您的站点已经有证书，那么可以忽略以下步骤。

配置证书过程

  • 首先通过 openssl genrsa -des3 -out ca.key 1024 来生成私钥。
  • 生成证书文件 openssl req -new -x509 -days 365 -key ca.key -out ca.crt。（这里需要注意的是，如果你的代理地址使用是域名形式或者IP形式）那么 Common Name 一项必须跟你的域名或则IP一致。
  • 删除私钥的密码 openssl rsa -in ca.key -out ca.key。
  • 生成客户端证书并导入计算机。 openssl pkcs12 -export -out ca.pfx -inkey ca.key -in ca.crt

证书最好使用认证机构生成的证书，因为方便么，不需要每台电脑安装一次。

使用方法

  • 执行 spdyproxy -k ca.key -c ca.crt -p 8080 >\dev\null 2>&1 &
  • 在浏览器端（Chrome）配置 PAC 如下即可。
